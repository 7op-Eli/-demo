# 物业管理系统 · 开发文档导航

本目录是项目的开发文档体系，服务于**多模型协作开发**。任何模型（DS / Codex / Claude）开始工作前，请先按下表读对应文档。

## 📚 文档清单

| 文档 | 谁维护 | 作用 | 何时读 / 写 |
|---|---|---|---|
| [主开发文档.md](主开发文档.md) | **DS 主维护**，所有人改完代码都回写 | 项目「单一事实来源」：架构、约定与坑、当前状态、本地运行、待办、DS 同步区 | DS **每次开工前必读**；改完代码同步「当前状态/待办」 |
| [开发历程与修复记录.md](开发历程与修复记录.md) | **次开发（Codex / Claude）撰写** | 按时间倒序记录开发历程、Bug 根因与修复、给 DS 的评审建议 | 排查/修复后**追加一条**；DS 可读「评审建议」领任务 |
| 本文件 README.md | 任何人 | 文档导航 + 协作规范 | 初次进入项目时 |

## 👥 角色分工

- **DS（DeepSeek）= 主开发**：负责功能实现与推进，维护《主开发文档》，在「DS 同步区」记录当前进度。
- **Codex / Claude = 次开发**：负责排查、修复、代码评审，撰写《开发历程与修复记录》，给 DS 留可执行建议。

## 🔄 协作规范

1. **改代码 → 同步文档**：任何模型改动后，
   - 影响架构/约定/状态 → 更新《主开发文档》对应小节；
   - 是一次修复/排查 → 在《开发历程与修复记录》**追加一条**（用文件末尾模板）。
2. **历程只追加不删**：保留完整开发史，最新记录放最上面。
3. **建议闭环**：次开发的建议写进《开发历程与修复记录》末尾「评审建议」；DS 处理后到《主开发文档》第 8 节「待办」勾掉。
4. **防回归红线**：《主开发文档》第 5 节「关键约定与坑」是高频踩雷点（尤其 context-path 与 Security 路径匹配），**改相关代码前必读**。

## 🚀 快速上手

- 本地启动 / 账号 / 端口：见《主开发文档》第 7 节（`admin / admin123`，前端 http://localhost:3000）。
- 最近做了什么 / 还有什么坑：见《开发历程与修复记录》最新一条 + 「评审建议」。
