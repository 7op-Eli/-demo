## 4. 接口模块与前缀

> **后端 context-path = `/api`**（见 `application.yml`）。所有 URL 实际带 `/api` 前缀。

| 前缀 | Controller | 鉴权 |
|---|---|---|
| `/auth` | AuthController | `/auth/login` 放行；`/auth/me` 需登录 |
| `/admin` | AdminController | **类级 `@PreAuthorize("hasRole('ADMIN')")`** |
| `/butler` | ButlerController | 读放行给登录用户；写需 EMPLOYEE/ADMIN |
| `/convenience` | ConvenienceController | 写需 ADMIN/EMPLOYEE |
| `/notices` | NoticeController | 读放行；`/notices/public/**` 完全公开；写需 ADMIN |
| `/repair` | RepairController | 多数需 EMPLOYEE/ADMIN |
| `/visitor` | VisitorController | 多数需 EMPLOYEE/ADMIN |
| `/fee` | PropertyFeeController | 写需 ADMIN/EMPLOYEE |

角色（`Constants`）：`1=业主 ROLE_OWNER` · `2=员工 ROLE_EMPLOYEE` · `3=管理员 ROLE_ADMIN`。

## 5. 关键约定与「坑」（⚠️ 改动前必读，防回归）

1. **context-path = `/api` 的双重含义**
   - Spring Security 的 `antMatchers(...)` 匹配的是**去掉 `/api` 之后**的路径 → 写 `/auth/login`，**绝不能写 `/api/auth/login`**（否则 permitAll 失效 → 登录 403）。
   - `JwtAuthenticationFilter` 用 `request.getRequestURI()`，它**包含 `/api`** → 这里判断要带 `/api`（如 `/api/auth/login`）。
   - 👉 这是本项目历史上登录 403 的根因，详见《开发历程与修复记录》。

2. **统一响应体**：`Result{ code, msg, data }`，成功 `code=200`。
   前端 axios 拦截器：`code===200` 时直接返回 `res.data`；否则弹出 `msg`。

3. **分页约定**：**1-based**（页码从 1 开始）。后端 `PageRequest.of(page-1, size)`，默认 `page=1`。
   分页体 `PageResult{ total, page, size, list }`。
   ⚠️ 传 `page=0` 会报 400「Page index must not be less than zero」。
   ⚠️ 已知问题：`PageResult` 自带 `code/msg` 导致**双层包裹**，待清理（见待办）。

4. **鉴权**：登录返回 JWT，前端放 `Authorization: Bearer <token>`。
   `SecurityConfig` 为 `anyRequest().authenticated()` + 方法级 `@PreAuthorize`。
   ⚠️ 未做「数据归属校验」——见待办中的越权(IDOR)项。

5. **密码**：BCrypt（`$2a$10$...`）。默认管理员 `admin / admin123`。
   ⚠️ 自动建号默认密码 `123456`、用户名=手机号。

6. **`ddl-auto: none`**：Hibernate 不建表、不校验 → **`schema.sql` 必须与实体字段严格一致**，改实体务必同步改 SQL。`open-in-view: false` → 懒加载只能在事务/Service 内触发。
